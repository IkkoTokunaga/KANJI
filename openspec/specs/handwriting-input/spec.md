# handwriting-input Specification

## Purpose

タッチ・ペン・マウス入力に対応した手書き用の描画キャンバスを提供し、キャンバスのクリア機能を備え、ユーザーの筆跡を採点に使える画像データとして取り出せるようにする。

## Requirements

### Requirement: 描画キャンバスでの手書き入力
システムは手書き用の描画キャンバスを提供し、ポインタの軌跡に沿って線を描画しなければならない（SHALL）。タッチ／ペン入力を主対象とし、PC のマウス入力でも同様に描画できなければならない（SHALL）。

#### Scenario: 線を描く
- **WHEN** ユーザーがキャンバス上でポインタを押下して移動する
- **THEN** その軌跡に沿って線が描画される

#### Scenario: タッチ操作での描画
- **WHEN** ユーザーがタッチ対応端末でキャンバスを指でなぞる
- **THEN** 指の軌跡に沿って線が描画される
- **AND** 描画中はページのスクロールが発生しない

### Requirement: キャンバスのクリア
システムは描画内容を消去する手段を提供しなければならない（SHALL）。

#### Scenario: 手動クリア
- **WHEN** ユーザーが「消す」操作を行う
- **THEN** キャンバスの描画内容がすべて消去される

#### Scenario: 次問への遷移時のクリア
- **WHEN** 次の問題へ進む
- **THEN** キャンバスの描画内容が自動的に消去される

### Requirement: 採点用画像の取り出し
システムは現在の手書き筆跡を採点に使える画像として取り出せなければならない（SHALL）。

#### Scenario: 手書き画像の取得
- **WHEN** 採点が要求される
- **THEN** キャンバスの描画内容が画像データとして取得できる

#### Scenario: 未記入での採点
- **WHEN** 何も描かれていない状態で採点が要求される
- **THEN** 類似率0%相当として扱われ、採点処理がエラーにならない
