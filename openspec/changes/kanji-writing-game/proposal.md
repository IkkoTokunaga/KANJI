## Why

子どもが漢字の「読み」と「書き」を結びつけて楽しく練習できる教材が欲しい。読みを聞いて／見て、例文の文脈の中で対象漢字を手書きする体験は、ドリル形式よりも記憶に残りやすい。まずは小学1年生で習う漢字を対象に、出題から採点までが一通り動くプロトタイプを作る。

## What Changes

- 漢字クイズの出題フローを追加する。対象漢字の読みを提示し、例文中の対象漢字を `□`（虫食い）で表示する。
- 手書き入力エリア（描画キャンバス）を追加する。ユーザーが対象漢字を手書きし、その筆跡を画像として取得できる。
- 採点機能を追加する。手書き画像とお手本フォントの漢字画像を画像類似度で比較し、類似率に応じてポイントを加算する。
- 例文の音声読み上げを追加する。ブラウザ内蔵の Web Speech API（`SpeechSynthesis`、`ja-JP`）で例文を読み上げる。
- 小学1年生漢字のコンテンツデータを追加する。MVP では数問（小1漢字のうち 10〜20 字程度）に各読み・例文・対象漢字位置を持たせる。

## Capabilities

### New Capabilities
- `kanji-quiz`: 1問分の出題と進行。読みの提示、例文の虫食い表示（対象漢字を `□` に置換）、回答後の正解表示と次問への遷移、累計ポイントの管理。
- `handwriting-input`: 描画キャンバス上での手書き入力。線の描画・消去（クリア）・1文字分の筆跡を採点用の画像として取り出す。
- `similarity-scoring`: 手書き画像とお手本漢字画像の画像類似度を算出し、類似率（0〜100%）とそれに基づく獲得ポイントを返す。
- `sentence-audio`: 例文テキストを Web Speech API で日本語音声として読み上げる。再生ボタンと未対応ブラウザ時のフォールバック挙動を含む。
- `kanji-dataset`: 小学1年生漢字の問題データ（漢字・読み・例文・例文内の対象漢字位置）を提供するデータ構造とサンプルデータ。

### Modified Capabilities
（なし。新規プロジェクトのため既存 spec の変更はない）

## Impact

- コード: `src/app`（Next.js App Router）に新規ページ／コンポーネントを追加。クイズ画面、手書きキャンバス、採点ロジック、音声再生、漢字データを配置する。
- 依存: 追加の外部依存は最小限を想定。手書きは HTML Canvas、音声はブラウザ標準の Web Speech API を利用するため新規 npm 依存は原則不要。
- ブラウザ: Canvas（手書き）と `SpeechSynthesis`（音声）に依存。音声未対応環境ではテキスト表示にフォールバックする。
- 対象範囲: MVP は小1漢字の数問。データ構造は字数拡張を見据えた形にする。
